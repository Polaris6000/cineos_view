### 포인트+쿠폰 결합 0원 결제 시 500 에러

##### 재현 방법

고객이 쿠폰 + 포인트를 합산해서 결제금액을 0원으로 만든 뒤 결제
POST /api/payment/confirm → 500

##### 원인
PaymentController.savePaymentInfo()에서 프론트가 보낸 bonusPolicyId로 적립 정책을 조회하는데, 
프론트가 이 값을 얻으려면 GET /api/admin/bonus-policy/list를 호출해야 함.
근데 이 경로는 SecurityConfig에서 /api/admin/** → JWT 필수로 막혀 있어서 고객 결제 화면에서는 401 반환. 
그래서 프론트가 fallback으로 bonusPolicyId=1을 보냈는데 해당 ID가 DB에 없어서 500 터진 것.


수정 위치 — PaymentController.java savePaymentInfo() 안
```
// 기존 — 프론트에서 받은 ID로 조회 (고객은 이 ID를 가져올 방법이 없음)
long bonusPolicyId = requestData.get("bonusPolicyId").asLong();
BonusPolicyDTO bonusPolicy = bonusPolicyService.getBonusPolicy(bonusPolicyId);

// 수정 — 서버가 직접 활성 정책 조회
BonusPolicyDTO bonusPolicy = bonusPolicyService.getActiveBonusPolicy();
getActiveBonusPolicy() 구현은 아래 3군데 추가 필요:

```

BonusPolicyRepository — findFirstByActivationTrueAndEndAtAfter(LocalDateTime now)
BonusPolicyService — getActiveBonusPolicy() 선언
BonusPolicyServiceImpl — getActiveBonusPolicy() 구현

1. BonusPolicyRepository.java
```
/**
 * activation = true 이고 endAt이 현재 시각 이후인 정책 중 id 오름차순 첫 번째 조회
 *
 * Spring Data JPA 네이밍 룰 자동 구현:
 *   findFirst          → LIMIT 1
 *   ByActivationTrue   → WHERE activation = 1
 *   AndEndAtAfter(now) → AND end_at > :now
 *
 * @param now 현재 시각 (LocalDateTime.now() 전달)
 * @return 활성 적립 정책 Entity (없으면 Optional.empty())
 */
Optional<BonusPolicyEntity> findFirstByActivationTrueAndEndAtAfter(LocalDateTime now);

```
2. BonusPolicyService.java
```
/**
 * 현재 활성화된 적립 정책 단일 조회
 * 결제 확인 시 bonusPolicyId를 프론트에서 받지 않고 서버가 직접 결정하기 위해 사용
 *
 * @return 활성 적립 정책 DTO
 * @throws NoSuchElementException 활성 정책이 없을 경우
 */
BonusPolicyDTO getActiveBonusPolicy();

```

3. BonusPolicyServiceImpl.java
```
/**
 * 현재 활성화된 적립 정책 단일 조회
 *
 * [사용 목적]
 * PaymentController.savePaymentInfo()에서 포인트 적립률 계산 시 호출.
 * 기존에는 프론트엔드가 bonusPolicyId를 직접 골라 요청 바디에 담아 보냈는데,
 * 고객 결제 화면은 JWT가 없어 /api/admin/bonus-policy/list (JWT 필수)에
 * 접근할 수 없어 401 → fallback ID가 DB에 없으면 500이 발생하던 문제를 해결.
 *
 * [조회 조건]
 *   - activation = true  : 관리자가 활성화 처리한 정책
 *   - endAt > 현재 시각  : 아직 만료되지 않은 정책
 *   - LIMIT 1            : 조건을 만족하는 정책 중 id가 가장 작은(먼저 등록된) 것 1개
 *
 * [예외 처리]
 * 활성 정책이 하나도 없으면 NoSuchElementException 발생.
 * → 관리자가 적립 정책을 등록·활성화하지 않은 상태에서 결제가 시도되면
 *   결제 확인 단계에서 500 에러로 이어지므로, 운영 전 반드시 정책 등록 필요.
 *
 * @return 현재 활성화된 적립 정책 DTO
 * @throws java.util.NoSuchElementException 활성 정책이 없을 경우
 */
@Override
public BonusPolicyDTO getActiveBonusPolicy() {
    return bonusPolicyRepository
            .findFirstByActivationTrueAndEndAtAfter(LocalDateTime.now())
            .map(BonusPolicyEntity::toDTO)
            .orElseThrow(() -> new NoSuchElementException("활성화된 적립 정책이 없습니다."));
}

```
4. PaymentController.java — savePaymentInfo() 내부
```

//   → 고객 결제 화면은 JWT가 없어 /api/admin/bonus-policy/list 에 접근 불가 (401)
//   → fallback으로 보내던 ID=1이 DB에 없으면 getBonusPolicy()에서 500 발생
//
// long bonusPolicyId = requestData.get("bonusPolicyId").asLong();
// BonusPolicyDTO bonusPolicy = bonusPolicyService.getBonusPolicy(bonusPolicyId);
// ──────────────────────────────────────────────────────────────────────────

// [변경 후] 서버가 직접 현재 활성 정책 조회
//   → 프론트엔드의 bonusPolicyId 의존성 제거
//   → activation=true 이고 endAt이 현재 시각 이후인 정책을 자동 선택
//   → 활성 정책이 없으면 NoSuchElementException → 500 (운영 전 정책 등록 필수)
BonusPolicyDTO bonusPolicy = bonusPolicyService.getActiveBonusPolicy();

```
